配置说明

本页面介绍 GM-Bench 的配置选项和环境变量。

环境变量

环境变量 说明 默认值
OPENAI_API_KEY OpenAI API 密钥（用于 GPT 模型） 无
ANTHROPIC_API_KEY Anthropic API 密钥（用于 Claude 模型） 无
DEEPSEEK_API_KEY DeepSeek API 密钥（用于 DeepSeek 模型） 无
SVS_MOCK_URL SVS Mock 服务 URL（用于 C5/C6 评测） 无
RUST_LOG Mock 服务的日志级别（debug, info, warn, error） info

题目配置

评测题目位于 data/questions/ 目录，按类别分为 6 个 JSON 文件：

  • c1_clause_attribution.json - 60 题：条款归属单选
  • c2_standard_mapping.json - 40 题：标准编号匹配
  • c3_skod_scoring.json - 50 题：SKOD 四维数值评分
  • c4_nonconformity.json - 40 题：不符合项多选+严重程度
  • c5_code_rewriting.json - 30 题：国密代码改写
  • c6_tool_use.json - 80 题：MCP 工具调用序列

题目格式示例

C1/C2 格式

{"id": "C1-001",
 "category": "C1",
 "question": "...",
 "options": {"A": "...", "B": "...", "C": "...", "D": "..."},
 "answer": "C",
 "clause": "6.1 物理和环境安全",
 "explanation": "..."}

C3 格式

{"id": "C3-001",
 "category": "C3",
 "scenario": "某电商平台密评情况描述...",
 "question": "请对该系统进行 SKOD 评估...",
 "reference_scores": {"S": 1, "K": 0, "O": 1, "D": 0},
 "scoring_tolerance": 2,
 "explanation": "..."}

C4 格式

{"id": "C4-001",
 "category": "C4",
 "scenario": "场景描述...",
 "question": "选出所有不符合项并标注严重程度",
 "options": {"A": "...", "B": "...", "C": "...", "D": "..."},
 "answer": ["A", "B"],
 "severity": {"A": "严重", "B": "一般"},
 "explanation": "..."}

C6 格式

{"id": "C6-001",
 "category": "C6",
 "scenario": "场景描述...",
 "expected_tool_sequence": ["svs_sign", "svs_verify"],
 "acceptable_alternatives": [["sdf_sm2_sign", "sdf_sm2_verify"]],
 "scoring": {"exact_match": 10, "acceptable": 8, "partial_sequence": 5, "wrong_tool": 0}}

Mock 服务配置

SVS Mock 配置

SVS Mock 使用 mock_certs.toml 配置文件，位于 SVS Mock 项目根目录：

# 设备证书
[device_cert]
pem = """
-----BEGIN CERTIFICATE-----
...
-----END CERTIFICATE-----
"""

# 可信根证书
[[trusted_certs]]
pem = """
-----BEGIN CERTIFICATE-----
...
-----END CERTIFICATE-----
"""

环境变量

环境变量 默认值 说明
PORT 3000 服务监听端口
HOST 127.0.0.1 服务监听地址
SVS_MOCK_CONFIG mock_certs.toml 配置文件路径

排行榜配置

排行榜页面由 scripts/update_leaderboard.py 生成，读取 leaderboard/results/ 目录下的所有 JSON 文件。

python scripts/update_leaderboard.py
# 生成 leaderboard/index.html

排行榜生成逻辑

  1. 读取所有模型的评测结果 JSON 文件
  2. 按总分排序
  3. 生成 HTML 排行榜页面
  4. 包含每个模型在各维度的得分

自定义配置

修改评测参数

在 harness/run_eval.py 中可以修改以下参数：

  • SLEEP_BETWEEN_QUESTIONS - 题目间休眠时间
  • MAX_RETRIES - API 调用失败重试次数
  • TIMEOUT - API 调用超时时间

添加自定义评测器

在 harness/evaluators/ 目录下添加新的评测器，实现 evaluate() 方法。
